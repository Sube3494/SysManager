# 📖 使用示例

## 🎯 基本用法

```
你: /sys ls
Bot: [/home/user]
     file1.txt
     file2.log
     README.md

你: /sys cd /var/log
Bot: 切换到: /var/log

你: /sys ls
Bot: [/var/log]
     nginx/
     syslog
     auth.log
```

---

## 📂 目录管理

### 切换目录

```
/sys cd /var/log      # 绝对路径
/sys cd nginx         # 相对路径
/sys cd ..            # 上级目录
/sys cd ~             # 用户主目录
/sys cd /             # 根目录
```

### 查看和重置

```
/sys pwd              # 显示当前目录
/sys reset            # 重置回初始目录
```

```
     1. ls
```

## 💪 强制全部输出

### 普通执行（自动截断）

```
你: /sys cat large.log
Bot: [/var/log]
     line 1
     line 2
     ...
     (已截断，完整10000字符，使用 -f 强制全部输出)
```

### 使用 -f 参数

```
你: /sys cat large.log -f
Bot: [/var/log]
     line 1
     line 2
     ... (全部10000字符都显示)
```

---

## 🚫 交互式命令处理

### 被拦截的命令

```
你: /sys vim config.txt
Bot: 不支持 vim (查看文件)
     替代: cat <文件>

你: /sys top
Bot: 不支持 top (查看进程)
     替代: top -bn1 | head -20 或 ps aux
```

### 正确用法

```
你: /sys cat config.txt
你: /sys top -bn1 | head -20
你: /sys ps aux
```

```

     nginx (CPU:0.5% MEM:1.2%)
```

## 💡 实用技巧

### 1. 批量操作

连续执行多个命令：
```
/sys cd /var/log/nginx
/sys tail -100 access.log
/sys grep "404" access.log
/sys wc -l access.log
```

### 2. 查找文件

```
/sys find /home -name "*.log"
/sys locate config.txt
/sys which python
```

### 3. 监控系统

```
/sys df -h
/sys free -h
/sys ps aux
/sys netstat -antp
/sys uptime
```

### 4. 日志查看

```
/sys tail -100 /var/log/syslog
/sys grep "error" /var/log/nginx/error.log
/sys journalctl -n 50
```

---

## ⚠️ 注意事项

1. **每条命令都显示路径** - `[/current/path]`
2. **命令前必须加 `/sys`** - 这是固定的命令前缀
3. **-f 参数慎用** - 可能导致超长输出
4. **cd 命令会保持目录状态** - 每个用户独立记录
